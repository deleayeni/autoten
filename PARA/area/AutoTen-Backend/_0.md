# AutoTen Backend — Area

## What this Area tracks

The **server-side plumbing** of AutoTen: Next.js server boundaries, API routes, server actions, configuration, and third-party integrations beyond the database.

This is the answer to: "what runs on the server, what external services does it talk to, and what's the glue in between?"

## Honest current state

AutoTen has **almost no dedicated backend code** right now. The site is:

- A server-rendered Next.js app (so "backend" = async server components calling Supabase)
- A single data-layer file ([src/data/cars.ts](../../../src/data/cars.ts))
- Two Supabase client helpers ([server.ts](../../../src/lib/supabase/server.ts), [client.ts](../../../src/lib/supabase/client.ts))

**What doesn't exist yet** (but the roadmap calls for):

- ❌ API routes (`src/app/api/**` is empty)
- ❌ Server actions (no `src/actions/` folder)
- ❌ Middleware (no `src/middleware.ts`)
- ❌ Auth — Supabase Auth is scaffolded in [server.ts](../../../src/lib/supabase/server.ts) but not wired to UI
- ❌ Payment webhooks (Paystack / Flutterwave)
- ❌ Email sending (Resend / SendGrid)
- ❌ SMS (Termii)
- ❌ WhatsApp Business API
- ❌ Scheduled jobs / cron

So this Area is mostly a **map of what's missing**, plus documentation of the thin layer that exists.

## Known issues (surfaced during this scan)

- **Supabase hostname is hardcoded** in [next.config.ts:10](../../../next.config.ts#L10) as `mswknjxlvcfhkbrulymt.supabase.co`. This is the **old paused project's hostname**. When you restore to a new Supabase project, this file must be updated or `<Image>` components will fail to load car images.
