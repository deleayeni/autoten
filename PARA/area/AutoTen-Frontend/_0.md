# AutoTen Frontend — Area

## What this Area tracks

A **living snapshot** of the AutoTen website as it exists today: what pages exist, what components they use, how styling is organized, what assets live in `public/`.

This is not a Resource (learning material). It's an inventory — the kind of doc you read to answer "what did I already build?" and "where is X?".

## Keeping this current

**Rule:** these files go stale every time you ship a feature. Don't pretend to maintain them by hand.

**Instead:** every few weeks (or after a significant change), ask me:
> "Rescan the repo and rewrite Area/AutoTen-Frontend."

I'll diff against the current files and update only what changed.
