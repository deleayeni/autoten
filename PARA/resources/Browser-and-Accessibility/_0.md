# Browser APIs and Accessibility

## What this folder covers

The raw browser/DOM APIs you used (IntersectionObserver, getComputedStyle), plus accessibility fundamentals — semantic HTML and ARIA attributes — that make the site usable for screen readers and keyboard users.

## Why it matters for this project

Your NavBar's transparent-over-hero behavior is powered by `IntersectionObserver`, a browser API — not React, not Next.js. Understanding where the framework ends and the browser begins makes you a much better web developer.

Accessibility matters because: (1) it's legally required in many markets, (2) your SEO depends on it, (3) it's the mark of a professional site.
