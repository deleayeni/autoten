# Web Security

## What this folder covers

Security concerns specific to a public-facing Next.js + Supabase site: environment variable hygiene, the anon-key-in-browser model, dependency CVEs, and how RLS is what keeps a "public" database from being a disaster.

## Why it matters for this project

AutoTen ships your Supabase anon key to every browser that visits the site. That's by design — but it only works because RLS is correctly configured server-side. The recent [React Server Components CVE fix](../../../README.md) in your git history is a real-world reminder that frameworks have security holes too.

Your DevOps background gives you a head start here, but frontend security has its own patterns.

## From DevOps, to web

The underlying principles (least privilege, defense in depth) are the same. What changes is the specific attack surface: now it's the browser.
